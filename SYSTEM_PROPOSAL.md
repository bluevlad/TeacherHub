# TeacherHub 시스템 제안서

## 1. 개요
**TeacherHub**는 학원 강사들이 자신(또는 특정 키워드)과 관련된 게시글을 주요 커뮤니티(네이버 카페, 다음 카페, 공무원 공부 갤러리 등)에서 자동으로 수집하고, AI를 통해 긍정/부정 여부를 분석하여 일일 현황을 파악할 수 있는 통합 대시보드 시스템입니다.

---

## 2. 주요 기능

### 2.1 데이터 수집 (Data Collection)
- **대상 사이트**: 
  - 네이버, 다음 카페 (공드림, 취업뽀개기 등)
  - 디시인사이드 (공무원 공부 갤러리 등)
- **기능**:
  - 지정된 키워드(강사명, 과목명 등)로 주기적(예: 1시간, 1일) 자동 검색 및 크롤링
  - 게시글 제목, 본문, 작성시간, 조회수, 댓글 수 수집
  - 댓글 내용 수집 (심층 분석용)

### 2.2 감성 분석 (Sentiment Analysis)
- **AI 엔진**:
  - 수집된 텍스트(제목+본문+댓글)에 대한 자연어 처리 (NLP)
  - 한국어 특화 감성 분석 모델 적용 (Positive/Negative/Neutral 판별)
  - 주요 키워드 추출 (Word Cloud)

### 2.3 대시보드 & 리포팅 (Dashboard)
- **일일 요약**: 오늘 수집된 게시글 수, 긍정/부정 비율 차트 제공
- **트렌드 분석**: 최근 1주일/1개월 간의 언급량 변화 그래프
- **실시간 알림**: 부정적 여론 급증 시 알림(Email/SMS)
- **원문 이동**: 분석 결과에서 클릭 시 해당 게시글 원문으로 링크

## 3. 시스템 아키텍처 (Architecture)

### 3.1 기술 스택 (Tech Stack)

| 구분 | 기술 스택 | 설명 |
|---|---|---|
| **Frontend** | React, Material UI | 사용자 대시보드 및 리포트 시각화 (기존 프로젝트와 통일성 유지) |
| **Backend** | Spring Boot | REST API 서버, 데이터 제공 및 사용자 관리 |
| **Crawler/AI** | Python (Playwright, Hugging Face) | 동적 웹 크롤링 및 NLP 감성 분석 수행 |
| **Database** | PostgreSQL | 수집 데이터 및 분석 결과 저장 |
| **Schedule** | Airflow or Spring Scheduler | 주기적 크롤링작업 관리 |

### 3.2 데이터 흐름 (Data Flow)
1. **Collector (Python)**: 사용자가 등록한 키워드로 각 커뮤니티 크롤링 수행
2. **Analyzer (Python)**: 수집된 Raw Data를 NLP 모델로 분석하여 감성 점수(Score) 산출
3. **Database**: 결과 데이터(게시글 정보 + 감성 점수) DB 적재
4. **Backend (Spring)**: API 요청에 따라 통계 데이터 가공 및 전달
5. **Frontend (React)**: 차트 및 리스트 형태로 시각화하여 사용자에게 제공

## 4. 구현 단계 (Roadmap)

### Phase 1: 기본 수집기 구축
- 네이버/다음 카페/디시인사이드 각 사이트별 크롤러 모듈 개발 (로그인 세션 처리 포함)
- Python 기반 스크립트로 CLI 형태 실행

### Phase 2: 감성 분석 모델 연동
- KoELELCTRA 등 사전학습된 한국어 모델을 활용하여 Fine-tuning 혹은 제로샷 분류 적용
- 긍정/부정 분류 정확도 테스트 및 개선

### Phase 3: 웹 서비스 통합
- Spring Boot 백엔드 API 개발 (키워드 등록, 조회)
- React 프론트엔드 대시보드 개발 (차트 연동)
- 자동화 스케줄러 설정
